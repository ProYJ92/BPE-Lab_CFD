OpenFOAM 사용 설명서: 검증 및 확인 (V&V)

OpenFOAM을 사용하여 얻은 CFD 시뮬레이션 결과의 신뢰성을 평가하고 엔지니어링 결정에 활용하기 위해서는 체계적인 검증(Verification) 및 확인(Validation), 즉 V&V 과정이 필수적입니다. OpenFOAM의 특성상 사용자가 직접 많은 부분을 설정하고 제어하므로, V&V의 중요성은 더욱 큽니다.

V&V의 일반적인 이론 및 절차에 대해서는 CFD 이론: 후처리 - V&V 섹션 또는 Ansys Fluent: V&V 섹션을 참고하십시오. 본 페이지에서는 OpenFOAM 환경에서의 V&V 수행 시 특별히 고려할 사항들을 중심으로 설명합니다.

1. 검증 (Verification) in OpenFOAM

OpenFOAM 솔버가 수학적 모델을 정확하게 풀고 있는지 확인하는 과정입니다.

  • 격자 수렴성 테스트 (Grid Convergence Study):

    OpenFOAM에서도 격자 품질과 밀도는 결과에 큰 영향을 미칩니다. blockMesh, snappyHexMesh 또는 외부 격자 생성 도구를 사용하여 최소 3가지 이상의 체계적으로 다른 밀도의 격자를 생성합니다. 각 격자에 대해 동일한 system 및 constant 디렉토리 설정을 사용하여 해석을 수행하고, 주요 물리량(예: forces, probes 함수 객체로 추출)을 비교하여 격자 독립성을 평가합니다. GCI (Grid Convergence Index) 분석을 수행할 수 있습니다.

    (팁: 격자 크기 변경 시 blockMeshDict의 블록별 셀 수, snappyHexMeshDict의 세분화 레벨 등을 조절)
  • 시간 간격 수렴성 테스트 (Time Step Convergence Study):

    비정상 상태 솔버(pisoFoam, pimpleFoam 등) 사용 시, controlDict의 deltaT 값을 변경해가며 해석을 수행하고 결과의 시간 간격 의존성을 평가합니다. 특히 pimpleFoam의 경우 CFL 수 기반의 자동 시간 간격 조절(adjustTimeStep) 기능 사용 시 최대 CFL 수(maxCo) 값에 따른 민감도 분석도 고려할 수 있습니다.

  • 이산화 스킴(Discretization Scheme) 민감도 분석:

    fvSchemes 파일에서 대류항(divSchemes) 등의 이산화 스킴을 변경(예: upwind vs. linearUpwind vs. limitedLinear)하면서 결과의 변화를 관찰합니다. 고차 정확도 스킴 사용 시 안정성 문제 발생 가능성도 염두에 둡니다.

  • 솔버 및 알고리즘 설정 민감도 분석:

    fvSolution 파일에서 선형 솔버의 허용 오차(tolerance), 압력-속도 연계 알고리즘(PISO, SIMPLE)의 설정값, 감속 계수(relaxationFactors) 등을 변경하며 결과의 민감도를 확인할 수 있습니다. (특히 사용자 정의 솔버 개발 시 중요)

  • 튜토리얼 및 벤치마크 사례 활용:

    OpenFOAM과 함께 제공되는 튜토리얼 사례(예: cavity, damBreak)를 직접 실행해보고, 이미 검증된 결과와 비교하여 자신의 설정 및 환경이 올바른지 간접적으로 확인할 수 있습니다.

2. 확인 (Validation) in OpenFOAM

OpenFOAM 시뮬레이션 결과가 실제 물리 현상을 정확하게 예측하는지 평가하는 과정입니다.

  • 실험 데이터와의 정량적 비교:

    가장 중요한 확인 절차입니다. 신뢰할 수 있는 실험 데이터와 OpenFOAM 결과를 비교합니다.

    1. 실험에서 사용된 형상과 동일한 격자를 OpenFOAM에서 생성합니다.
    2. 실험 조건(유입 속도, 온도, 물성치 등)을 0 디렉토리의 경계 조건 파일 및 constant 디렉토리의 물성 파일에 정확하게 반영합니다.
    3. controlDict의 functions 항목에 probes, sets, surfaces 등을 정의하여 실험 데이터와 동일한 위치/영역에서 데이터를 추출합니다.
    4. 추출된 데이터를 Python(Matplotlib, Pandas), Gnuplot 등을 사용하여 실험 데이터와 함께 그래프로 비교하고 오차를 분석합니다.
    5. 필요시 turbulenceProperties의 난류 모델, fvSchemes의 이산화 기법 등을 조정하여 모델을 개선합니다.
    (예시: Python 스크립트를 사용하여 OpenFOAM 결과와 실험 데이터를 플롯팅하는 코드 일부 또는 비교 그래프)
  • 이론해/분석해와의 비교:

    단순화된 문제에 대해 OpenFOAM으로 해석하고, 그 결과를 이론해와 비교합니다. (예: Hagen-Poiseuille 유동)

  • 다른 CFD 코드 결과와의 비교 (Cross-Code Validation):

    동일한 문제에 대해 신뢰성 있는 상용 CFD 소프트웨어(예: Ansys Fluent)로 해석한 결과와 OpenFOAM 결과를 비교할 수 있습니다. 이 경우, 두 코드 간의 물리 모델, 수치 기법, 격자 등의 차이점을 명확히 인지하고 비교해야 합니다.
