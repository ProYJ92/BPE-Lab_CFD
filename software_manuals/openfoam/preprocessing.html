OpenFOAM 사용 설명서: 전처리

OpenFOAM의 전처리 과정은 주로 격자 생성(Meshing)과 사례 설정(Case Setup)으로 구성됩니다. OpenFOAM은 텍스트 기반의 딕셔너리 파일들을 통해 모든 설정을 관리하므로, 파일 구조와 각 파일의 역할에 대한 이해가 중요합니다.

1. 격자 생성 (Meshing)

OpenFOAM은 자체 격자 생성 유틸리티와 외부 격자 변환 도구를 제공합니다.

blockMesh

단순한 형상(주로 직육면체 블록 조합)에 대한 정형(structured) 또는 블록-구조(block-structured) 격자를 생성하는 유틸리티입니다. system/blockMeshDict 파일에 형상 정보(정점, 모서리, 블록, 경계면)와 격자 밀도를 정의합니다.

blockMesh

단순하지만 강력하며, 기본적인 유동 문제나 복잡한 격자의 배경 격자 생성에 사용됩니다.

(이미지: blockMeshDict 파일 예시 또는 blockMesh로 생성된 단순 격자)

snappyHexMesh

복잡한 형상(STL 또는 OBJ 파일 기반) 주위에 비정형 육면체 지배적 격자를 자동으로 생성하는 강력한 유틸리티입니다. 배경 격자(주로 blockMesh로 생성), 형상 표면, 그리고 다양한 격자 제어 파라미터(세분화 레벨, 경계층 설정 등)를 system/snappyHexMeshDict 파일에 정의합니다.

snappyHexMesh -overwrite

품질 좋은 격자를 얻기 위해서는 설정 파일에 대한 이해와 여러 번의 시도가 필요할 수 있습니다.

(이미지: snappyHexMeshDict 파일 일부 또는 snappyHexMesh로 생성된 복잡 형상 격자)

외부 격자 생성 도구 활용

Salome, Gmsh, Ansys Meshing, Pointwise 등 다양한 외부 GUI 기반 격자 생성 소프트웨어를 사용하여 격자를 생성한 후, OpenFOAM 형식으로 변환하여 사용할 수 있습니다. 각 소프트웨어에서 지원하는 내보내기 형식(예: CGNS, MED, UNV, MSH)에 따라 OpenFOAM 유틸리티(예: fluentMeshToFoam, gmshToFoam, ideasUnvToFoam)를 사용하여 변환합니다.

# 예시: Fluent .msh 파일을 OpenFOAM 형식으로 변환
fluentMeshToFoam meshfile.msh

격자 품질 확인

생성된 격자의 품질은 checkMesh 유틸리티를 사용하여 확인할 수 있습니다. 왜곡도(skewness), 종횡비(aspect ratio), 면적(face area), 최소/최대 각도, 음수 체적(negative volume) 등을 검사하여 보고합니다.

checkMesh

격자 품질이 나쁘면 해석 결과의 정확도가 낮아지거나 발산할 수 있으므로, checkMesh 결과를 바탕으로 격자를 수정해야 합니다.

2. 사례 설정 (Case Setup)

OpenFOAM 사례는 일반적으로 특정 디렉토리 구조를 따릅니다. 주요 디렉토리는 다음과 같습니다:

  • 0 (또는 초기 시간) 디렉토리: 초기 조건 및 경계 조건 파일을 포함합니다. 각 변수(예: U (속도), p (압력), T (온도), k (난류 운동 에너지))에 대한 파일이 위치합니다.
  • constant 디렉토리: 시간에 따라 변하지 않는 파일들을 포함합니다.
    • polyMesh: 격자 정보 파일들이 위치하는 하위 디렉토리. (points, faces, owner, neighbour, boundary 등)
    • transportProperties: 유체의 물성치(점성, 밀도 등)를 정의하는 파일.
    • turbulenceProperties: 난류 모델 선택 및 관련 설정을 하는 파일.
    • 기타 물리 모델 관련 딕셔너리 파일 (예: RASProperties, LESProperties, thermophysicalProperties).
  • system 디렉토리: 해석 제어 및 솔버 설정을 위한 파일들을 포함합니다.
    • controlDict: 해석 시작/종료 시간, 시간 간격, 데이터 저장 빈도, 라이브러리 로드 등 전반적인 제어 설정을 담당합니다.
    • fvSchemes: 유한 체적법의 이산화 기법(차분 스킴)을 정의합니다 (예: 시간항, 구배항, 발산항, 라플라시안항).
    • fvSolution: 선형 대수 방정식 시스템의 솔버, 허용 오차, 감속 계수, 압력-속도 연계 알고리즘(PISO, SIMPLE 등) 등을 설정합니다.
    • 격자 생성 관련 딕셔너리 파일 (예: blockMeshDict, snappyHexMeshDict).
(이미지: OpenFOAM 사례 디렉토리 구조 예시)

각 딕셔너리 파일은 고유한 문법과 키워드를 가지므로, OpenFOAM 사용자 가이드나 튜토리얼을 참조하여 정확하게 작성해야 합니다. 기존 튜토리얼 사례를 복사하여 수정하는 방식으로 시작하는 것이 일반적입니다.
