Ansys Fluent 사용 설명서: 검증 및 확인 (V&V)

CFD 시뮬레이션 결과의 신뢰성과 정확성을 확보하기 위해서는 체계적인 검증(Verification) 및 확인(Validation) 과정, 즉 V&V가 필수적입니다. V&V는 Ansys Fluent를 포함한 모든 CFD 해석에서 중요한 단계이며, 엔지니어링 의사 결정에 시뮬레이션 결과를 활용하기 위한 전제 조건입니다.

상세한 이론적 배경은 CFD 이론: 후처리 - V&V 섹션을 참고하십시오.

1. 검증 (Verification)

"코드가 의도한 수학적 방정식을 올바르게 풀고 있는가?" 에 대한 질문에 답하는 과정입니다. 즉, 수치 알고리즘과 코드 구현의 정확성을 평가합니다. Fluent 사용자 관점에서의 주요 검증 활동은 다음과 같습니다.

  • 격자 수렴성 테스트 (Grid Convergence Study / Grid Independence Test):

    이산화 오류(Discretization Error)를 평가하고, 사용된 격자가 결과에 미치는 영향을 확인하는 가장 중요한 검증 절차입니다.

    1. 최소 3가지 이상의 체계적으로 다른 밀도(예: Coarse, Medium, Fine)의 격자를 준비합니다. 각 격자 단계별 셀 수의 비율을 일정하게 유지하는 것이 좋습니다 (예: 선형 치수 기준 1.5~2배).
    2. 각 격자에서 동일한 물리 모델 및 경계 조건으로 Fluent 해석을 수행합니다.
    3. 주요 관심 물리량(예: 항력 계수, 특정 지점의 압력, 최대 속도)을 각 격자 결과에서 추출합니다.
    4. 격자 밀도 변화에 따른 관심 물리량의 변화를 그래프로 나타내거나, Richardson Extrapolation과 같은 기법을 사용하여 격자 수렴 지수(Grid Convergence Index, GCI)를 계산하여 이산화 오차를 정량화합니다.
    5. 결과가 격자 밀도에 더 이상 민감하게 변하지 않는 "격자 독립적인(grid-independent)" 해를 얻었는지 판단합니다.
    (이미지: 격자 밀도에 따른 결과 변화 그래프 또는 GCI 계산 예시)
  • 시간 간격 수렴성 테스트 (Time Step Convergence Study):

    비정상 상태(Transient) 해석의 경우, 시간 간격(Time Step Size)이 결과에 미치는 영향을 평가합니다. 격자 수렴성 테스트와 유사하게 여러 다른 시간 간격을 사용하여 결과를 비교합니다.

  • 반복 수렴 오류 확인:

    솔버 설정에서 잔차(Residuals)가 충분히 낮은 수준(예: 10-5 이하)으로 감소했는지, 그리고 모니터링하는 주요 물리량이 안정화되었는지 확인하여 반복 수렴 오류가 무시할 만큼 작은지 평가합니다.

  • 코드 간 비교 (Code-to-Code Comparison):

    가능하다면, 동일한 문제에 대해 다른 신뢰할 수 있는 CFD 코드(예: OpenFOAM) 또는 이론해/수치해 벤치마크 데이터와 결과를 비교합니다.

2. 확인 (Validation)

"올바른 수학적 방정식을 풀고 있는가?" 또는 "시뮬레이션 결과가 실제 물리 현상을 정확하게 예측하는가?" 에 대한 질문에 답하는 과정입니다. 즉, 시뮬레이션 모델이 실제 물리 현상을 얼마나 정확하게 모사하는지 평가합니다.

  • 실험 데이터와의 정량적 비교:

    가장 직접적이고 중요한 확인 방법입니다. 신뢰할 수 있는 고품질의 실험 데이터(예: 풍동 실험 결과, PIV 측정 데이터, 학술 논문에 발표된 실험값)와 Fluent 시뮬레이션 결과를 정량적으로 비교합니다.

    1. 실험 조건(형상, 유입 속도, 온도, 물성치 등)과 Fluent 시뮬레이션의 경계 조건 및 설정을 최대한 일치시킵니다.
    2. 실험에서 측정한 위치와 동일한 위치에서 시뮬레이션 데이터를 추출합니다.
    3. 결과를 그래프나 표로 비교하고, 오차(Error)와 불확실성(Uncertainty)을 분석합니다.
    4. 오차가 크다면 모델링 가정(예: 난류 모델, 경계 조건 단순화)의 타당성을 재검토하고 모델을 개선합니다.
    (이미지: 실험 데이터와 CFD 결과 비교 그래프)
  • 이론해 또는 분석해와의 비교:

    단순화된 문제에 대해 이론해 또는 분석해가 존재하는 경우, 이를 Fluent 결과와 비교하여 모델의 기본적인 정확성을 평가할 수 있습니다. (예: 완전 발달된 파이프 유동, 평판 위 경계층 유동)

  • 벤치마크 문제 활용:

    학계에서 널리 사용되는 표준 벤치마크 문제(예: Lid-driven cavity, Backward-facing step)에 대한 시뮬레이션을 수행하고, 발표된 다른 연구 결과들과 비교합니다.
